## 1. 목표 정의
본 파이프라인의 목표는 텍스트 입력(Briefing)을 기반으로 비디오/오디오 스크립트 초안 생성, 캡션 자동 생성, 배경 음악 추천을 자동화하는 것이다.

## 2. 기술 스택 (초기 제안)
- **언어:** Python (기존 블로그 자동화와의 연계를 위해)
- **핵심 라이브러리:** requests (API 통신), Pydantic (데이터 구조 정의), LangChain (LLM 연동 시)
- **외부 API:** (추후 선정)
- **데이터 흐름:** 입력 텍스트 $\rightarrow$ LLM 처리 $\rightarrow$ 영상/음성 스크립트 생성 $\rightarrow$ 최종 결과물 저장

## 3. 데이터 흐름 상세 (Flow)

1. **입력 단계 (Input):** 사용자는 주제(Topic)와 원하는 분위기(Tone)를 입력한다.
2. **처리 단계 (Processing):**
    *   입력된 주제를 기반으로 LLM을 호출하여 영상에 적합한 스크립트(영상 내레이션)를 생성한다.
    *   생성된 스크립트를 기반으로 텍스트를 음성 파일(TTS)로 변환한다.
3. **출력 단계 (Output):** 생성된 스크립트와 음성 파일을 저장한다.

## 4. 다음 단계 (Next Steps)

1. **API 선정:** 어떤 TTS(Text-to-Speech) 및 영상 생성 API를 사용할지 결정한다.
2. **구조 설계:** Pydantic 모델을 사용하여 입력/출력 데이터의 구조를 확정한다.
3. **구현:** 선정된 API를 연동하여 실제 코드를 작성한다.
